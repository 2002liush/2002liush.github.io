github + hexo在mac搭建自己博客记录

今天尝试了搭建自己的博客，遇到了许多的问题，特此记录一下。

1 环境搭建

我是根据这片博客搭建的：https://blog.csdn.net/m0_60880170/article/details/126716159

注意：博客里面的所有步骤都是需要在root用户下进行的。

问题一：在使用命令sudo hexo d部署到github或者可以说把工程提交到github的时候出现了问题：连接不到git仓库。

解决：这是由于生成的ssh密匙错误，如果我们使用ssh-keygen -t rsa -C “xxx@github.com”生成密匙那么生成的当前用户的，但是我们用的sudo hexo d部署是在root用户下的，所以会提示找不到仓库。

所以我们需要使用sudo ssh-keygen -t rsa -C “xxx@github.com”生成ssh密匙

这个问题的解决是参照这片博客：https://blog.csdn.net/Sheri_stx/article/details/111386802

2 思考

通过搭建完环境后，对hexo、github以及自己的博客之间的关系。

hexo是一个能够生成html网页的工具，用来在本地生成自己的博客页面，我们也可以在本地启动这个网页，命令是sudo hexo s，只不过这种启动方式需要我们始终在本地保持启动的状态，这显然是不方便的，因为我们电脑不可能一直开机。

所以github就担任了一个帮我们显示这个页面的功能，因为github仓库可以不显示为代码的形式而显示为html页面的形式。

在配置工程中弄明白了配置流程以及各个部件所起的作用，我觉得这是非常重要的。

3 如何添加新的博客文章

在配置博客环境的文件夹下，比如我的是在liushenghuan/software/blogs这个文件夹下配置的环境，在这个文件夹里的source/_posts/文件夹下存放着所有的md文章，在这个文件夹下的md文章在你生成博客页面的时候都会添加到你的博客中，所以我们只需要把md文件放到这个文件夹下然后生成上传就可以了。

具体步骤：

  1. 在电脑任意位置创建md文件
  2. 在blogs文件夹下使用sudo hexo new “文件名”创建一个md文件
  3. 把这个md文件内容复制到刚才创建的md文件中
  4. sudo hexo cl清除
  5. sudo hexo g生成工程
  6. sudo hexo d部署到github
  7. 部署到github上后，github更新页面会很慢，此时我们可以本地查看，使用sudo hexo s启动，然后本地查看页面是否符合自己的预期

注意：先把想要上传的文章写好后再复制到/source/_posts/文件夹下，后面修改文章不要在source/posts/文件夹下修改，养成良好的习惯。

5 修改butterfly主题

butterfly是一款开源的适合hexo的主题，配置方法在上面的博客也有。butterfly有许多主题选项可供用户选择，现在记录的就是如何自定义butterfly。

修改butterfly的方式是修个根目录下的_config.butterfly.yml文件，这个文件里面提供了butterfly主题的选项，用户可以通过修改这个文件来修改主题。
